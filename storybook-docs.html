‚öôÔ∏è Setup & Installation

Get Storybook running in your project with the required addons.

1
Install Storybook
Run the Storybook CLI initializer in your project root.
npx storybook@latest init
2
Install Required Addons
Add accessibility, interactions, and design token support.
npm install -D \
  @storybook/addon-a11y \
  @storybook/addon-interactions \
  @storybook/addon-designs \
  @chromatic-com/storybook
3
Configure .storybook/main.ts
Register addons and configure the framework.
import type { StorybookConfig } from '@storybook/react-vite'

const config: StorybookConfig = {
  stories: ['../src/**/*.stories.@(js|jsx|ts|tsx|mdx)'],
  addons: [
    '@storybook/addon-essentials',
    '@storybook/addon-a11y',
    '@storybook/addon-interactions',
    '@chromatic-com/storybook',
  ],
  framework: { name: '@storybook/react-vite' },
}

export default config
4
Create Your First Story
Stories use the CSF (Component Story Format) 3.0 standard.
import type { Meta, StoryObj } from '@storybook/react'
import { Button } from './Button'

const meta: Meta<typeof Button> = {
  component: Button,
  title: 'Components/Button',
}
export default meta

type Story = StoryObj<typeof Button>

export const Primary: Story = {
  args: { variant: 'primary', children: 'Click me' },
}

// Button.stories.tsx
import type { Meta, StoryObj } from '@storybook/react'
import { Button } from './Button'

const meta: Meta<typeof Button> = {
  title: 'Components/Button',
  component: Button,
  args: { children: 'Click me' },
  argTypes: {
    variant: {
      control: 'select',
      options: ['primary', 'secondary', 'ghost', 'danger'],
    },
    disabled: { control: 'boolean' },
  },
}
export default meta

export const Primary: Story = {
  args: { variant: 'primary' }
}
export const AllVariants: Story = {
  render: () => (
    <div style={{ display: 'flex', gap: '8px' }}>
      {['primary', 'secondary', 'ghost', 'danger'].map(v => (
        <Button key={v} variant={v}>{v}</Button>
      ))}
    </div>
  )
}
Prop Type Default Required Description
variant 'primary' | 'secondary' | 'ghost' | 'danger' 'primary' Visual style variant of the button
size 'sm' | 'md' | 'lg' 'md' Controls padding and font size
disabled boolean false Disables interaction and reduces opacity
loading boolean false Shows spinner and disables interaction
onClick () => void ‚Äî Click handler callback
children ReactNode ‚Äî required Button label or content

Icon-Only Buttons

‚úì Do
  • ‚úì Use clear, action-oriented labels ("Save changes")
  • ‚úì Use primary variant for the main CTA only
  • ‚úì Include loading state for async operations
  • ‚úì Keep button text concise ‚Äî 1‚Äì3 words ideal
  • ‚úì Pair icon + text for better recognition
‚úï Don't
  • ‚úï Use more than one primary button per view
  • ‚úï Use vague labels like "Click here" or "Submit"
  • ‚úï Disable without explaining why in nearby text
  • ‚úï Use danger variant for standard destructive flows
  • ‚úï Place icon after text (icons lead, text follows)

üöÄ Deploy to Chromatic

Chromatic provides visual regression testing and persistent Storybook hosting.

1
Add Chromatic to CI
# .github/workflows/chromatic.yml
name: Chromatic
on: [push]
jobs:
  chromatic:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - run: npm ci
      - uses: chromaui/action@latest
        with:
          projectToken: ${{ secrets.CHROMATIC_PROJECT_TOKEN }}
          buildScriptName: build-storybook
2
Netlify Alternative
# netlify.toml
[build]
  command = "npm run build-storybook"
  publish = "storybook-static"
